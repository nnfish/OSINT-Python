# OpenOSINT 互联网舆情采集分析系统 - 系统设计文档

## 1. 系统概述

### 1.1 项目背景

OpenOSINT 是一款面向政企及专业人员使用的网络舆情分析处理系统，可以实现网络上开源情报的采集、分析、监控、预警等功能。

### 1.2 系统目标

- 实现多源数据的自动化采集
- 提供智能化的情报分析能力
- 支持实时监控和预警通知
- 生成专业的分析报告

### 1.3 技术栈

| 层级 | 技术 |
|------|------|
| 前端 | Vue 3 + Element Plus |
| 后端 | Python + FastAPI |
| 数据库 | MySQL 8.0 |
| 部署 | Docker + Docker Compose |
| AI | Qwen3 / OpenAI 兼容 API |

## 3. 核心业务流程

### 3.1 情报采集流程

```
用户创建任务 → 配置关键词/数据源 → 启动采集 → 数据采集 → 存储入库
```

### 3.2 情报分析流程

```
选择数据范围 → 调用 LLM 分析 → 生成报告 → 存储/展示
```

### 3.3 情报监控流程

```
配置监控规则 → 定时检查 → 条件匹配 → 触发通知
```

### 3.4 情报预警流程

```
配置预警规则 → 实时监测 → 触发条件 → 生成预警 → 推送通知
```

## 5. 部署说明

### 5.1 Docker 部署

```bash
# 构建镜像
docker-compose build

# 启动服务
docker-compose up -d

# 查看日志
docker-compose logs -f

# 停止服务
docker-compose down
```

### 5.2 服务端口

| 服务 | 端口 |
|------|------|
| Nginx (前端) | 80 |
| FastAPI (后端) | 8000 |
| MySQL | 3306 |

### 5.3 环境要求

- Docker 20.10+
- Docker Compose 2.0+
- 内存：4GB+
- 存储：10GB+
